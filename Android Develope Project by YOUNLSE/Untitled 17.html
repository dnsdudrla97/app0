  • 버튼을 클릭했을 때는 chooseContacts() 메서드가 호출되되록 한다.
  • 그리고 chooseContacts() 메서드 안에서는 인텐트 객체를 만들고 startActivityForResult() 메서드를 호출하면서 파라미터로 전달한다.
  • 인텐트 객체 생성
    • 파라미터
      1. Intent.ACTION_PICK
        1. 액션 정보를 전달하고
      1. ContactsContract.Contacts.CONTENT_URI
        1. 전달한다. 이값은 연락처 정보를 조회하는데 사용되는 URI 값이다.
  • startActiviryForResult() 메서드가 호출되면 연락처를 선택할 수 있는 화면이 표시된다.
  • 사용자가 연락처를 하나 선택하면 onActivityForResult() 메서드가 자동으로 호출된다.

onActivityResult()

@Override
    protected void onActivityResult(int requestCode, int resultCode, @Nullable Intent data) {
        if (resultCode == RESULT_OK) {
            if (requestCode == 101) {
                try {
                    Uri contactsUri = data.getData();
                    String id = contactsUri.getLastPathSegment(); 
										// 선택한 연락처의 id 값을 확인하기

                    getContacts(id);

                } catch (Exception e) {
                    e.printStackTrace();
                }
            }
        }
    }
  • 이 메서드로 전달되는 인텐트 객체의 getData() 메서드를 호출하면 선택된 연락처 정보를 가리키는 Uri 객체가 반환된다.
  • 이 Uri 객체의getLastPathSegment() 메서드를 호출하여 id 값을 확인한다.
  • 이렇게 id 값을 확인하는 이유는 선택한 연락처의 상세 정보가 다른 곳에 저장되어 있기 때문이다.
  • getContacts라는 메서드를 호출하면서 id 값을 파라미터로 전달하도록 하고 getCount() 메서드를 추가한다.

getContacts()

public void getContacts(String id) {
        Cursor cursor = null;
        String name = "";

        try {
            cursor = getContentResolver().query(ContactsContract.Data.CONTENT_URI,
                    null,
                    ContactsContract.Data.CONTACT_ID + "=?",
                    new String[] { id },
                    null);
            // ContentResolver 객체의 query() 메서드 호출하기
            if (cursor.moveToFirst()) {
                name = cursor.getString(cursor.getColumnIndex(ContactsContract.Data.DISPLAY_NAME));
                println("Name : "+ name);

                String columns[] = cursor.getColumnNames();
                for (String column : columns) {
                    int index = cursor.getColumnIndex(column);
                    String columnOutput = ("#" + index + "-> [" + column + "] " + cursor.getString(index));
                    println(columnOutput);
                }
                cursor.close();
            }
        } catch (Exception e) {
            e.printStackTrace();
        }

    }
  • getContacts() 메서드 안에서는 ContentResolver 객체의 query() 메서드를 호출한다.
  • query() 메서드의 파라미터
    1. ContactsContract.Data.CONTENT_URI를 전달
      1. 이 Uri 값은 앞에 서 인텐트를 만들 때 넣어주었던 Uri 값과 다르다.
      1. 왜냐? 이 Uri 값은 연락처의 상세 정보를 조회하는 데 사용되는 Uri이기 때문이다.
    1. nulll
    1. ContactsContract.Data.CONTACT_ID + "=?"
      1. id 칼럼의 이름과 함께 =? 라는 글자를 붙여서 전달한다.
      1. 연락처의 상세 정보를 저장하는 테이블에는 많은 칼럼들이 있는데 그 칼럼의 이름중에서 id 칼럼의 이름은 ContactsContract.CONTACT_ID 상수로 확인할 수 있다.
  • query() 메서드가 반환하는 Cursor 객체를 참조해다,면 결과 레코드를 화면에 출력한다.
  • 먼저 연락처의 이름 칼럼 값을 출력한다.
  • 이름 칼럼은 ContractsContract.Data.DISPLAY_NAME 상수로 확인할 수 있으므로 이 칼러 이름을 이용해 칼럼의 인덱스 값을 찾고 그 인덱스 값을 이용해 이름 값을 알아낸다.
  • 알아낸 이름은 println() 메서드를 호출하여 출력한다.
  • 그 아래에서는 for 문을 이용해 모든 칼럼의 이름과 그 칼럼의 값을 출력한다.

연락처를 조회하려면 READ_CONTACTS, WRITE_CONTACTS 권한이 필요하다.

<uses-permission android:name="android.permission.READ_CONTACTS"/>
<uses-permission android:name="android.permission.WRITE_CONTACTS"/>

  • 하나의 연락처를 선택했을 때 그 연락처 안에 상당히 많은 칼럼들이 정의되어 있다는 것을 알 수 있다.
  • 코드를 통해 알게 된 것 처럼, 연락처에 대한 정보는 ContactsContract.Contacts 객체가 가리키는 테이블에 저장되어 있으며 연락처에 대한 상세 정보는 ContactsContract.Data 객체가 가리키는 테이블에 저장되어 있다.
  • 이외에도 ContactsContract.RawContacts 테이블도 있으나 위의 두 개 테이블에 대해서만 알아도 연락처를 조회할 수 있다.
