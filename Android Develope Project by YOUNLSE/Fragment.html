상단 탭 보여주기

  • 모바일 단말은 일반적으로 화면의 크기가 작기 떄문에 하나의 화면에 너무 많은 구성 요소를 넣으면 성능이나 사용성 면에서 좋지 않다.
  • 안드로이드 경우에도 하나의 화면을 나타내는 액티비티를 최대한 많이 분리시켜서 하나의 화면에 보이는 뷰의 개수를 줄여주는 것이 좋다.
  • 그러나 때로는 하나의 화면에 여러 가지 구성 요소를 넣어두고 필요할 때 전환하여 보여주는 게 좋을때도 있다.
    • 대표적인 것들로 서브 화면들이다. 예를 들어, 상단에 버튼이 두 개 있고 그 두개의 버튼을 누를 때마다 아래에 다른 화면을 보여주는 방식으로 만든다면 고객의 일반 정보와 신용도를 구분 하여 보여줄 수 있다.
    • 이처럼 한 명의 고객과 관련된 서로 다른 두 가지 정보를 한 화면에서 전환하여 보여줄 수 있으므로 더 직관적인 화면을 구성할 수 있다.

탭 뷰(Tab)

  • 몇 개의 버튼을 두고 그중 하나의 버튼을 눌러 서브 화면을 전환하는 방식처럼 하나의 뷰에서 여러 개의 정보를 볼 때 일반적으로 사용하는 뷰
  • 탭은 안드로이드의 전화번호부를 비롯한 몇 개의 기본 앱에서 볼 수 있는데 상단에 있는 탭을 누를 때마다 내용이 보이는 화면 영역이 전환되어 나타난다.
  • 탭은 내비게이션 위젯이라고 불리기도 하며 상단 탭과 하단 탭으로 구분할 수 있다.
  • 최근에는 하단 탭을 더 많이 사용한다.
  • 상단 탭의 경우에는 액션바에 탭 기능을 넣어 보여주는 방법으로 제공
  • 하단 탭은 별도의 위젯으로 제공

레이아웃에 추가

  • 메인 액티비티의 XML 레이아웃에 상단 타이틀 부분의 레이아웃을 직접 정의할 수 있다.
  • 자동으로 만들어지는 메인 액티비티의 레이아웃에는 타이틀 부분이 빠져있고 액션바가 들어간 스타일이 지정되어 있지만 액션바가 들어가 있지 않은 스타일로 바꾸는 경우에는 직접 상단의 타이틀 부분 레이아웃을 만들 수 있다.

XML 제작

<CoordinatorLayout>
	<AppBarLayout>
		<Toolbar>
		</Toolbar>
		<TabLayout>
		</TabLayout>
	</AppBarLayout>
	<FrameLayout>
	<FrameLayout>
</CoordinatorLayout>
  • 상단의 액션바 부분과 가운데 내용이 보이는 부분이 함께 들어간 구조에서는 두 부분의 크기가 잘 맞게 보여야 하므로 CoordinatorLayout이 사용된다.
  • 그리고 그 안에 AppBarLayout과 FrameLayout이 들어간다.
  • AppBarLayout이 상단의 액션바 영역, FrameLayout이 내용이 보이는 화면 부분이 된다
  • Toobar는 액션바 안에서 타이틀이 메뉴 아이콘이 보일 영역을 말하며 탭 버튼이 그 아래에 보일 수 있도록 TabLayout이 들어가 있다.
  • 탭 버튼을 보이지 않게 하고 싶다면 TabLayout을 넣지 않으면 된다.

CoordinatorLayout

  • 액션바 영역을 포함한 전체 화면의 위치를 잡아주는 역활을 하므로 가장 바깥에 위치하고 있다.
  • CoordinayotLayout 안에 AppBarLayout 과 함께 다른 레이아웃을 넣으면 그 둘 간의 간격이나 위치가 자동으로 결정된다.

AppBarLayout

  • 액션바를 가리키는데 이 안에는 Toolbar 가 들어갈 수 있으며, 탭을 사용하는 경우에는 탭의 버튼들이 들어갈 수 있는 TabLayout을 추가할 수 있다.
  • AppBarLayout 아래쪽에는 FramLayout을 넣어 화면의 내용을 구성할 수 있다.

속성 앞에 붙는 접두어

xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:app="http://schemas.android.com/apk/res-auto"
  • xmlns:app 대신 xmlns:myapp 으로 변경하고 싶다면 그 파일 안에서 참조하는 app도 모두 myapp으로 변경해야 한다.

TabLayout

  • tabMode의 값을 fixed
  • tabGravity의 값을 fill로 설정하여 [탭] 버튼들이 동일한 크기를 갖게 만든다.

FrameLayout

  • id 값은 container로 설정하여 자바 소스 코드에서 이 안에 프래그먼트르 넣을 수 있도록 한다.

액션바 속성 팁

actionBar.setDisplayHomeAsUpEnabled(false);			
//액션바 아이콘을 업 네비게이션 형태로 표시합니다.

actionBar.setDisplayShowTitleEnabled(false);		
//액션바에 표시되는 제목의 표시유무를 설정합니다.

actionBar.setDisplayShowHomeEnabled(false);			
//홈 아이콘을 숨김처리합니다.

탭 3개 전환 방법

  1. 탭은 세 개를 만들 것이다.
  1. 각각의 탭을 누를 때마다 하단에 다른 내용이 보여야 한다.
  1. 액티비티는 하나인데 그중에서 일부분만 화면이 변경되도록 해야 하므로 앞에서 살펴보았던 프래그먼트를 사용할 수 있다.
  1. FrameLayout으로 하단에 공간을 확보한 후 그 안에 상황에 따라 서로 다른 프래그먼트를 넣어줄 수 있다.
  1. 여기서는 FrameLayout 안에 세 개의 프래그먼트를 넣을 것이므로 먼저 세 개의 프래그먼트를 프로젝트 폴더의 res/layout 폴더 안에 추가한다.
  1. 첫 번째 프로그먼트를 위한 xml 레이아웃은 fragment1.xml 이름으로 만들고 최상위 레이아웃은 linearLayout으로 변경한 후 그 안에 버튼을 하나 추가한다.

Fragment1.java → 나머지 1,2,3 동일하게 inflate 션 한다.

public class Fragment1 extends Fragment {
    @Nullable
    @Override
    public View onCreateView(@NonNull LayoutInflater inflater, @Nullable ViewGroup container, @Nullable Bundle savedInstanceState) {
        return inflater.inflate(R.layout.fragment1, container, false);
    }
}

MainActivity.java

public class MainActivity extends AppCompatActivity {
    Toolbar toolbar;

    Fragment1 fragment1;
    Fragment2 fragment2;
    Fragment3 fragment3;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        toolbar = findViewById(R.id.toolbar);
        setSupportActionBar(toolbar);
        //Toolbar 객체는 코드에서 setSupportActionBar() 메서드를 사용해 액션바로 설정해야 한다.
        //setSupportActionBar() 메서드는 액티비티에 디폴트로 만들어진 액션바가 없을 경우에만 동작한다.
        //그런데, 프로젝트가 만들어질때 메인 액티비티에는 자동으로 액션바가 만들어진다. -> 이것은 테마(theme)를 액션바가 들어 있는 테바로 설정했기 때문이다.
        //styles.xml -> NoActionBar 로 변경한다. -> 이 스타일을 적용한 액티비티에는 액션바가 만들어지지 않는다.
        //따라서 setSupportActionBar() 메서드로 호출하여 직접 액션바를 설정해야 한다.
        //이미 액션바를 코드에서 설정했으므로 세 개의 프래그먼트 객체를 만들어 변수에 할당하고 첫 번째 프래그먼트가 화면에 보이도록 한다.

        ActionBar actionBar = getSupportActionBar();
        actionBar.setDisplayShowTitleEnabled(false);

        fragment1 = new Fragment1();
        fragment2 = new Fragment2();
        fragment3 = new Fragment3();

        getSupportFragmentManager().beginTransaction().replace(R.id.container, fragment1).commit();

        TabLayout tabs = findViewById(R.id.tabs);
        tabs.addTab(tabs.newTab().setText("Call Log"));
        tabs.addTab(tabs.newTab().setText("Spam log"));
        tabs.addTab(tabs.newTab().setText("CallBook"));

        //addTab() 메서드가 있어서 [탭] 버튼을 추가할수 있게 한다.
        //여기서는 세 개의 [탭] 버튼을 추가했으며 각각이 [탭] 버튼을 눌렀을 때 container라는 id를 가진 FrameLayout 안에 각각의 [탭] 버튼에 해당하는 프래그먼트 화면이 보이도록 한다.
        //OnTabSelectedListener -> 이 리스너는 [탭] 버튼이 선택될 때마다 그 리스너 안에 있는 onTabSelected() 메서드가 호출되도록 한다.
        //이 메서드로는 현재 선택된 탭 객체가 전달되므로 탭의 position 정보를 확인한후 각각의 fragment에 축한다.

        tabs.addOnTabSelectedListener(new TabLayout.OnTabSelectedListener() {
            @Override
            public void onTabSelected(TabLayout.Tab tab) {
                int position = tab.getPosition();
                Log.d("MainActivity", "selected Tab : " + position);

                Fragment selected = null;
                if (position == 0) {
                    selected = fragment1;
                } else if (position == 1) {
                    selected = fragment2;
                } else if (position == 2) {
                    selected = fragment3;
                }
                getSupportFragmentManager().beginTransaction().replace(R.id.container, selected).commit();
            }

            @Override
            public void onTabUnselected(TabLayout.Tab tab) { }

            @Override
            public void onTabReselected(TabLayout.Tab tab) { }
        });
    }
