나인패치

  • 이미지뷰를 XML 레이아웃에 추가해서 화면에 보여줄 때 이미지가 나타나는 영역보다 원본 이미지가 작으면 시스템이 이미지 크기를 자동으로 늘려준다.
  • 이 기능은 해상도가 서로 다른 단말에서도 일정한 비율로 이미지의 크기를 지정하면 이미지가 자동으로 크기에 맞게 늘어나거나 줄어들게 하므로 아주 유용한 기능이다.
  • 이런 나인패치 과정을 거치면 이미지의 일부분이 깨져 보이거나 왜곡이 발생하기 때문에 발생한 문제점을 해결할 방법이 필요하게 된다.
    • 예를 들면, 버튼의 경우에는 텍스트만 보여주거나 텍스트와 아이콘을 같이 보여주는 방식만을 사용하면 모르겠지만
    • 버튼 자체를 포토샵과 같은 그래픽 프로그램에서 만든 후 버튼에 표시하는 방식을 사용하면 버튼 이미지의 크기와 버튼의 크기가 같아야 하는 문제가 생긴다.
    • 앱을 만들 떄는 단말의 해상도에 상관없이 크기를 지정하는 경우가 많기 때문에 "100dp"와 같은 값으로 버튼의 가로 크기를 지정하였다면 실제 픽셀 수는 단말에 따라 달라질것이다.
    • 이 과정에서 버튼에 올린 이미지는 실제 크기와 달라지는데 그럴 경우 이미지의 일부가 왜곡되어 보이게 된다.

왜곡정도

  • 왼쪽 이미지는 버튼 모양을 그래픽으로 만든것이다. 그림의 파란색 부분은 투명 영역을 의미한다.
  • 만약 이미지의 크기가 옆으로 늘어나면 모서리 부분이 옆으로 늘어나면서 왜곡이 생긴다. 왜곡은 다른 부분보다. 특히 모서리 부분에 많이 생기는데 그 이유는 버튼 이미지의 모서리를 곡선 모양으로 처리했기 때문이다.

나인 패치(Nine Patch) 로 해결해보자

  • 이미지가 늘거나 줄거나 할때 생기는 이미지 왜곡을 해결하는 방법을 정의
  • 다음 사진은 나인패치를 적용한 것입니다. 각각의 이미지의 크기는 가로, 세로 각각 2px 커졌으며, 이미지의 가로와 세로 끝부분의 픽셀들은 흰색 또는 검은색으로 지정되어 있다.

  • 이렇게 수정한 이미지 파일 이름은 ooo.9.png 처럼 파일 확장자 앞에 '.9'를 붙여야 한다.
  • 안드로이드에서는 이 방식으로 이름을 지은 파일을 원본 이미지의 가로, 세로 끝부분의 픽셀을 모두 나인패치 이미지의 정보를 담은 것으로 인식한다.
  • 이 정보는 흰색인 경우 늘어나지 않는 영역, 검은색인 경우 늘어나는 영역으로 구분되는데 위의 그림과 같이 지정하면 가운데 부분만 늘어나므로 이미지의 크기가 커지더라도 모서리 부분은 원래의 이미지 모양을 유지할 수 있다.
  • 나인 패치 이미지를 만들고 싶다면 포토샵과 같은 그래픽 편집 툴을 사용하면 된다.
  • 가장 간단한 방법은 png, jpg와 같은 이미지보다 가로,세로 크기가 2픽셀씩 큰 이미지를 새로 만들고 원본 이미지를 가운데에 복사하면 된다. 그러면 가장자리 한 픽셀씩을 흰색 또는 검은색으로 바꿀 수 있다.

Nine Patch 적용/미적용

  • 버튼의 배경으로 이미지를 설정한 것처럼 자바 코드에서도 설정할 수 있다. 뷰의 배경색으로 색생과 이미지를 지정하는 메서들은
//[Reference]
void setBackgroundColor (int color)
void setBackgroundDrawable (Drawable d)
void setBackgroundResource (int resid)
  • setBackgroundResource() 메서드는 xml 레이아웃의 background 속성과 같으므로 xml 레이아웃에 추가된 각각의 버튼들은 이 속성 값으로 지정된 이미지를 배경으로 설정하고 text 속성으로 설정된 텍스트의 길이 갑셍 맞추어 이미지 크기가 자동으로 변경된다.
